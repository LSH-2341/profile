이수호

게임 시스템을 직접 구현하고 개선하는 Unity 클라이언트 개발자 지망생입니다.

Unity와 C#을 기반으로 플레이어 조작, 크리처 AI, 인벤토리, UI, 스폰 시스템 등 게임 플레이에 필요한 핵심 시스템을 구현해왔습니다. AI 도구를 활용해 코드 초안을 만들고, 프로젝트 구조에 맞게 직접 적용·수정·통합·테스트하며 실제 작동하는 시스템으로 완성하는 경험을 쌓고 있습니다.

02

크리처 AI, 소리 반응, 손전등 판정, 인벤토리, 수위 상승 등 게임 로직 구현 경험이 있습니다.

03

기능 구현에서 끝내지 않고 버그 원인을 분석하고 구조를 개선하는 과정을 중요하게 생각합니다.

Technical Highlights

상태 기반 크리처 AI

Deadline에서 크리처 행동을 순찰, 감지, 추격, 탐색, 복귀 상태로 분리하고, 플레이어의 시야, 손전등, 발소리, 마이크 입력에 따라 다른 반응을 수행하도록 구현했습니다.

EnemyRegistry & Target Search Optimization

VANTA에서 씬 전체 적을 반복 탐색하던 구조를 EnemyRegistry 방식으로 개선하고, 자동 공격, 번개 무기, E스킬이 활성 적 목록을 참조하도록 구성했습니다.

레벨업, 잠식률 & 로컬 랭킹

VANTA에서 경험치 누적 후 중복 없는 3개의 업그레이드 선택지를 제공하고, 선택 결과가 스탯, 스킬, 잠식률에 반영되도록 구현했습니다. 사망 후에는 생존 시간, 처치 수, 레벨을 기준으로 로컬 TOP 10 랭킹을 저장합니다.

인벤토리 & 퀵슬롯

Sinking Island에서 아이템 획득, 슬롯 저장, 수량 관리, 퀵슬롯 사용, 스탯 물약 적용 흐름이 하나로 이어지도록 인벤토리 시스템을 구현했습니다. 인벤토리가 가득 찬 경우 아이템이 사라지지 않도록 획득 성공 여부를 검사하는 구조로 개선했습니다.

높이 / 낮밤 기반 스폰

Sinking Island에서 Raycast와 맵 높이값을 활용해 자원과 몬스터가 실제 지형 위에 배치되도록 하고, 높이와 낮밤 조건에 따라 다른 프리팹이 등장하도록 구성했습니다. 스폰 위치 거리 비교에는 sqrMagnitude를 사용해 반복 연산을 줄였습니다.

수위 상승 & 침수 피드백

Sinking Island 시간에 따라 수위가 상승하고, 플레이어 위치와 수면 높이를 비교해 침수 상태를 판정했습니다. 침수 시 데미지, 화면 효과, UI 변화를 연결해 플레이어에게 위험 상태를 전달했습니다.

AI-Assisted Development

Development Process

프로젝트 개발 과정에서 AI 도구를 코드 작성 보조 도구로 활용했습니다. 구현하고 싶은 기능 요구사항과 오류 상황을 정리한 뒤, AI를 통해 코드 초안과 수정안을 생성하고 Unity 프로젝트에 적용했습니다.

이후 인스펙터 연결, 컴포넌트 배치, 실행 테스트, 오류 재검증을 반복하며 실제 게임 흐름에 맞게 기능을 통합했습니다.

Applied Areas

  • Deadline: 크리처 AI, 소리 감지, 손전등 수리, 소화기 탈출, 정신력 시스템 적용
  • Sinking Island: 인벤토리, 퀵슬롯, 랜덤 스탯 물약, 수위 상승, 높이 / 낮밤 기반 스폰 적용
  • VANTA: 자동 공격, 레벨업 선택지, 번개 무기, E스킬, 대시, 보스, 로컬 랭킹 적용
  • VANTA의 EnemyRegistry 구조 개선과 범위 스킬 데미지 누락 문제 해결 과정 검증

My Responsibility

AI가 생성한 코드를 그대로 사용하는 것이 아니라, 프로젝트의 요구사항과 오류 상황을 정리하고 생성된 코드가 Unity 환경에서 정상적으로 동작하는지 확인했습니다.

기능 간 충돌이나 실행 오류가 발생하면 문제 상황을 다시 정리해 수정안을 생성하고, 이를 Unity 프로젝트에 적용·통합·테스트하는 방식으로 개발을 진행했습니다.

What I Learned

AI를 활용하면서 단순히 코드를 생성하는 것보다, 기능 요구사항을 명확히 정리하고 오류 상황을 구체적으로 설명하는 과정이 중요하다는 점을 배웠습니다.

이를 통해 Unity 컴포넌트 연결, C# 스크립트 구조, 시스템 간 데이터 흐름, 실행 테스트와 오류 재검증의 중요성을 경험했습니다.

Problem Solving

Unity 빌드 및 개발 환경 문제 해결

플랫폼 전환과 빌드 과정에서 발생한 오류를 코드 문제와 환경 문제로 나누어 확인했습니다. 프로젝트 폴더 정리, 모듈 확인, 에디터 재설치, 다른 PC 테스트를 통해 원인을 좁혀가며 빌드 문제를 해결했습니다.

크리처 AI 반응 로직 개선

Deadline의 크리처가 단순히 플레이어를 따라오는 구조에 그치지 않도록, 소리, 시야, 손전등, 마이크 입력 조건에 따라 다른 반응을 보이도록 상태 기반 로직으로 분리했습니다.

인벤토리와 성장 흐름 안정화

Sinking Island에서 아이템 획득, 슬롯 저장, 퀵슬롯 사용, 랜덤 스탯 물약 적용이 하나의 흐름으로 이어지도록 시스템을 연결했습니다. 또한 인벤토리가 가득 찬 상태에서 아이템이 사라지지 않도록, 아이템 추가 성공 여부를 확인한 뒤 오브젝트를 삭제하는 방식으로 개선했습니다.

EnemyRegistry와 범위 스킬 문제 해결

VANTA에서 공격과 스킬 로직이 씬 전체 적을 반복 탐색하던 구조를 EnemyRegistry 방식으로 개선했습니다. 또한 범위 스킬 적용 중 적이 사망하면서 목록이 변경되어 일부 적이 데미지를 받지 않는 문제를, 임시 타겟 리스트를 사용해 해결했습니다.
