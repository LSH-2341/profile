Project Overview

Game Concept

Deadline은 과제 제출을 앞둔 학생이 학교 건물 안에서 깨어나, 층별 퍼즐과 크리처의 위협을 피해 5층에서 1층까지 내려가 탈출하는 1인칭 공포 탈출 게임입니다.

플레이어는 각 층에서 단서와 아이템을 찾고, 퍼즐을 해결하며 다음 층으로 이동합니다. 단순히 길을 찾는 것이 아니라 크리처의 특성을 파악하고, 정신력과 아이템을 관리해야 합니다.

Player Goal

최종 목표는 1층에 도착해 마지막 문을 열고 건물 밖으로 탈출하는 것입니다. 크리처에게 잡히면 정신력이 1칸 감소하며, 정신력이 모두 소진되면 게임이 처음부터 다시 시작됩니다.

맵에 존재하는 드링크를 획득하면 정신력을 1칸 회복할 수 있고, 소화기를 들고 있을 경우 잡힌 순간 스페이스 입력으로 위기 상황에서 벗어날 수 있습니다.

Core Systems

Floor-based Puzzle Progression

게임은 5층에서 시작해 각 층의 퍼즐을 해결한 뒤 다음 층으로 이동하는 구조로 설계했습니다. 플레이어는 키패드, 단서, 아이템을 활용해 층별 진행 조건을 해결해야 합니다.

Creature AI System

크리처마다 다른 감지 조건과 대응 방식을 적용했습니다. 보이지 않는 크리처는 전등 깜박임과 발소리로 위치를 유추하게 만들었고, 시점형 크리처는 손전등과 플레이어 시야 조건을 함께 사용하도록 구성했습니다. 소리 감지 크리처는 발소리, 키패드 오답, 마이크 입력 등으로 발생한 소리에 반응합니다.

Sanity & Recovery System

플레이어가 크리처에게 잡히면 정신력이 1칸 감소하고, 정신력이 모두 소진되면 처음부터 다시 시작됩니다. 맵에 배치된 드링크 아이템을 획득하면 정신력이 1칸 회복되어, 플레이어가 회피뿐만 아니라 회복 아이템 탐색까지 고려하도록 설계했습니다.

Flashlight Repair System

손전등은 일정 시간마다 고장나도록 설정했습니다. 고장 시 플레이어는 1, 2, 3, 4 중 무작위로 제시되는 숫자 입력을 순서대로 성공해야 손전등을 다시 사용할 수 있습니다.

Fire Extinguisher Escape System

소화기는 모든 크리처에게 공통으로 적용되는 위기 탈출 수단입니다. 플레이어가 소화기를 들고 있는 상태에서 잡히면 스페이스 입력으로 소화기를 발사하고, 해당 크리처는 잠시 사라졌다가 일정 시간 후 다시 등장합니다.

Interaction UI System

아이템 획득, 키패드 조작, 손전등 수리, 소화기 사용, 크리처에게 잡힌 상황 등 플레이어가 현재 할 수 있는 행동을 UI로 안내하도록 구성했습니다.

Data Structure & Algorithm

State-based Creature Logic

크리처 행동을 단순 추격이 아니라 순찰, 감지, 추격, 탐색, 복귀 상태로 분리했습니다. 각 상태는 플레이어와의 거리, 소리 발생 여부, 시야 조건, 손전등 상태에 따라 전환됩니다.

Noise Event Handling

플레이어의 발소리, 키패드 오답, 마이크 입력 등에서 발생한 소리를 하나의 소리 이벤트로 처리하고, 소리 감지 크리처가 해당 위치를 추적하도록 구성했습니다.

View & Flashlight Detection

시점형 크리처는 플레이어 카메라가 크리처를 바라보는지와, 손전등 빛이 크리처에게 닿는지를 함께 검사합니다. 두 조건이 동시에 만족될 때만 크리처가 멈추도록 설계했습니다.

Random Input Repair Logic

손전등 고장 시 1, 2, 3, 4 중 무작위 숫자를 순서대로 생성하고, 플레이어가 해당 입력을 성공하면 손전등이 복구되도록 구현했습니다.

Sanity Count Management

크리처에게 잡힐 때마다 정신력 카운트를 감소시키고, 드링크 획득 시 정신력을 1칸 회복하도록 처리했습니다. 정신력이 모두 소진되면 게임이 처음부터 다시 시작됩니다.

Common Escape Logic

크리처 종류와 관계없이 플레이어가 소화기를 들고 있으면 공통 탈출 로직을 실행합니다. 잡힌 순간 스페이스 입력을 감지하고, 성공 시 크리처를 비활성화한 뒤 일정 시간 후 재등장시킵니다.

Problem Solving

Creature Reaction Design

처음에는 크리처가 단순히 플레이어를 따라오는 구조에 가까웠지만, 공포게임에서 긴장감을 만들기에는 부족했습니다. 이를 해결하기 위해 크리처마다 다른 감지 조건을 부여했습니다.

보이지 않는 크리처는 전등 변화와 발소리로 위치를 유추하게 만들고, 시점형 크리처는 손전등과 플레이어 시야를 동시에 요구하도록 구성했습니다. 소리 감지 크리처는 플레이어 행동에서 발생한 소리에 반응하도록 개선했습니다.

Flashlight Failure & Repair

시점형 크리처는 손전등으로 대응할 수 있기 때문에, 손전등이 항상 정상 작동하면 긴장감이 줄어드는 문제가 있었습니다.

이를 보완하기 위해 손전등이 일정 시간마다 고장나도록 설정하고, 무작위 숫자 입력을 통해 수리하는 로직을 추가했습니다. 그 결과 플레이어가 크리처 대응과 손전등 관리 사이에서 긴장감을 느끼도록 만들었습니다.

Catch Recovery System

크리처에게 잡히는 순간이 곧바로 실패로 이어지면 플레이 경험이 너무 단조로워질 수 있었습니다. 그래서 정신력 시스템과 소화기 탈출 시스템을 함께 적용했습니다.

플레이어는 잡히면 정신력이 감소하지만, 소화기를 들고 있다면 스페이스 입력으로 크리처를 밀어내고 잠시 생존할 수 있습니다. 또한 드링크를 통해 감소한 정신력을 회복할 수 있게 하여 탐색의 의미를 강화했습니다.

Sound-based Tension

소리 감지 크리처는 플레이어가 이동하거나 퍼즐을 틀렸을 때 발생하는 소리를 위험 요소로 연결하기 위해 설계했습니다.

발소리, 키패드 오답, 마이크 입력을 소리 이벤트로 처리하여, 플레이어가 이동 방식과 퍼즐 입력에 더 신중해지도록 만들었습니다.

Result & Improvement

현재 Deadline은 PC 플랫폼을 기준으로 개발 중이며, 플레이어 이동, 상호작용, 키패드 퍼즐, 손전등, 크리처 AI, 마이크 소음, UI 시스템 등 핵심 플레이 시스템을 구현했습니다.

향후에는 VR 환경으로 확장할 수 있도록 상호작용 방식과 아이템 조작 방식을 개선하고, 크리처별 행동 패턴과 층별 공포 연출을 강화할 예정입니다.
