담당 역할

게임 기획, 시스템 설계, Unity 클라이언트 개발, UI 구현, 전투 시스템, 성장 시스템, 적 스폰 및 난이도 상승 구조를 개인으로 개발했습니다.

Core Systems

Enemy Spawn & Difficulty System

생존 시간과 잠식률에 따라 적 스폰 간격, 최대 적 수, 적 능력치가 변화하도록 구성했습니다. 플레이어가 강해질수록 게임의 위험도 함께 증가하는 구조를 만들었습니다.

EXP & Level Up Choice System

몬스터 처치 후 경험치 구슬을 획득하고, 레벨업 시 3개의 랜덤 업그레이드 선택지를 제공하여 매 플레이마다 다른 성장 흐름을 만들 수 있도록 구현했습니다.

Skill & Auto Attack System

가장 가까운 적을 탐색해 자동으로 투사체를 발사하고, E스킬과 번개 무기를 통해 범위 공격과 자동 타격이 가능하도록 구성했습니다.

System Flow

Core Game Loop

VANTA의 기본 플레이 흐름은 전투, 성장, 위험 증가, 재도전으로 이어지도록 설계했습니다. 플레이어가 강해질수록 잠식률이 상승하고, 그에 따라 적의 압박도 함께 증가합니다. (잠식률: VANTA 게임의 난이도 상승 요소)

EnemyRegistry Flow

기존에는 공격, 스킬, 스폰 제한에서 씬 전체의 적을 반복 탐색했습니다. 이를 개선하기 위해 적이 생성될 때 등록되고, 사망하거나 사라질 때 제거되는 EnemyRegistry 구조를 도입했습니다.

Data Structure & Algorithm

EnemyRegistry 기반 적 목록 관리

기존에는 자동 공격, 번개 무기, 스킬, 스폰 제한에서 FindObjectsOfType 기반으로 씬 전체의 적을 탐색했습니다. 몬스터 수가 증가하는 생존 액션 게임 구조에 맞게 EnemyRegistry를 도입하여 살아있는 적 목록을 관리하도록 개선했습니다.

거리 기반 타겟 탐색

자동 공격은 등록된 적 목록 중 플레이어와 가장 가까운 적을 탐색하여 공격 대상으로 지정합니다. 반복적인 거리 비교에서는 Vector3.Distance 대신 sqrMagnitude를 사용하여 불필요한 제곱근 계산을 줄였습니다.

중복 없는 랜덤 업그레이드 선택

전체 업그레이드 후보 목록에서 무작위 선택지를 뽑고, 이미 선택된 항목은 후보에서 제거하여 레벨업 시 중복 없는 3개의 선택지가 등장하도록 구현했습니다.

잠식률 기반 난이도 조절

플레이어가 강력한 선택지를 고를수록 잠식률이 상승하고, 잠식률이 적 스폰 간격과 최대 적 수에 영향을 주도록 구성했습니다. 이를 통해 성장과 위험이 동시에 증가하는 게임 흐름을 만들었습니다.

Problem Solving

FindObjectsOfType 기반 적 탐색 구조 개선

초기 구현에서는 공격이나 스킬 사용 시마다 씬 전체의 적을 탐색했습니다. 적 수가 계속 증가하는 게임 구조에서는 비효율적일 수 있다고 판단하여, 적이 생성될 때 등록되고 사라질 때 제거되는 EnemyRegistry 구조로 개선했습니다.

범위 스킬 데미지 누락 문제 해결

EnemyRegistry 개선 후 범위 스킬 사용 시 일부 적만 데미지를 받는 문제가 발생했습니다. 원인은 데미지 적용 중 적이 사망하면서 Registry 목록에서 제거되고, 반복문 인덱스가 밀려 일부 적이 검사되지 않는 것이었습니다.

이를 해결하기 위해 먼저 범위 안의 적을 임시 타겟 리스트에 저장한 뒤, 해당 리스트를 기준으로 데미지를 적용하도록 수정했습니다. 이를 통해 리스트 순회 중 데이터 변경으로 인한 누락 문제를 방지했습니다.

Result & Improvement

현재 VANTA는 프로토타입 완성을 목표로 개발 중이며, 기본 전투, 적 스폰, 경험치 획득, 레벨업 선택지, 스킬, 대시, 번개 무기 시스템을 구현했습니다.

향후에는 몬스터와 투사체 생성에 Object Pooling을 적용하고, 보스 패턴, 균열 이벤트, 랭킹 시스템을 추가하여 반복 플레이 구조를 강화할 예정입니다.
