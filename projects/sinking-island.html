The Sinking Island

타임어택 생존 탈출 게임

시간이 지날수록 물에 잠기는 섬에서 자원을 수집하고, 점점 위로 이동하며 강력한 몬스터를 피해 제한 시간 안에 탈출하는 1인용 타임어택 생존 탈출 게임입니다.

Project Overview

프로젝트 정보

  • 프로젝트명: The Sinking Island
  • 장르: 1인용 타임어택 생존 탈출 게임
  • 개발 기간: 2025.09 ~ 2025.11.25
  • 개발 형태: 팀 프로젝트 / 5명
  • 담당 역할: 팀장
  • 플랫폼: PC
  • 개발 환경: Unity, C#

담당 역할

팀장으로서 보고서 작성과 게임의 전체 기획을 담당했습니다. 물이 점차 차오르며 플레이어가 위로 이동할 수밖에 없는 구조를 기획했고, 위로 갈수록 강력한 몬스터가 등장하도록 설계하여 플레이가 지루하지 않고 지속적인 긴장감을 유지하도록 구성했습니다.

Core Systems

Inventory & Quick Slot System

인벤토리와 퀵슬롯을 분리하여 아이템 획득, 스택 처리, 슬롯 교환, 손 장착, 소비 아이템 사용 흐름이 하나로 이어지도록 구현했습니다.

Water Rising System

시간이 지남에 따라 물의 높이가 상승하고, 플레이어가 수면 아래에 위치하면 침수 상태로 판정하여 지속 데미지와 화면 효과가 적용되도록 구현했습니다.

Height-based Resource Spawn

맵의 높이에 따라 서로 다른 자원과 몬스터가 자동으로 배치되도록 구현했습니다. 인스펙터에서 프리팹만 추가하면 자동으로 스폰 목록에 반영될 수 있도록 구성했습니다.

Player Status UI

플레이어의 체력과 상태에 따라 UI가 변화하도록 구현했습니다. 체력 감소, 저체력 상태, 침수 상태가 화면 효과와 HUD에 반영되도록 연결했습니다.

Random Stat Potion

몬스터 처치 시 드롭되는 물약 아이템을 통해 플레이어의 스탯 중 일부가 랜덤으로 증가하도록 구현했습니다.

Sound System

아이템 사용, 플레이어 피격, 몬스터 등장, 환경 변화 등 주요 게임 이벤트에 사운드를 연결하여 생존 상황의 몰입감을 높였습니다.

Overall Game Flow

Sinking Island는 가장 낮은 해변가에서 시작해 몬스터를 처치하며 성장하고, 절벽과 길을 따라 점점 높은 층으로 이동하는 생존 탈출 게임입니다. 층이 높아질수록 몬스터의 스탯이 증가하며, 각 층의 보스를 처치해야 마지막 층의 보트를 수리하고 섬을 탈출할 수 있습니다.

Data Structure & Algorithm

Slot-based Inventory Structure

고정 슬롯 기반의 인벤토리와 퀵슬롯 구조를 사용하여 아이템 데이터와 수량을 슬롯 단위로 관리했습니다. 아이템 획득 시 기존 스택을 먼저 확인하고, 빈 슬롯이 있을 경우 새 슬롯에 저장되도록 구성했습니다.

Raycast 기반 지형 탐색

자원 생성 시 임의 위치를 정한 뒤 위에서 아래로 Raycast를 사용하여 실제 지형 위의 유효한 생성 위치를 찾도록 구현했습니다. 이를 통해 공중이나 잘못된 위치에 자원이 생성되는 문제를 줄였습니다.

높이 기반 프리팹 선택

생성 위치의 Y값을 기준으로 높이 구간을 판단하고, 해당 구간에 맞는 자원 또는 몬스터 프리팹이 등장하도록 구성했습니다. 이를 통해 플레이어가 위로 올라갈수록 다른 자원과 위협을 경험하도록 만들었습니다.

겹침 방지 스폰 로직

생성된 자원의 위치를 저장하고, 새로운 자원 생성 시 기존 위치와의 거리를 비교하여 일정 거리 이상 떨어진 곳에만 배치되도록 구현했습니다. 이를 통해 자원이 한곳에 뭉쳐 생성되는 문제를 방지했습니다.

시간 기반 수위 상승

매 프레임 물 오브젝트의 높이를 증가시키고, 플레이어의 위치와 물의 높이를 비교하여 침수 여부를 판단했습니다. 이 구조를 통해 시간 제한과 이동 압박을 게임 플레이에 직접 연결했습니다.

Problem Solving

생존 압박 구조 설계

단순히 자원을 수집하는 생존 게임은 플레이 흐름이 느슨해질 수 있다고 판단했습니다. 이를 해결하기 위해 시간이 지날수록 섬이 물에 잠기고, 플레이어가 더 높은 지역으로 이동할 수밖에 없는 구조를 설계했습니다.

자원 자동 배치 구조 구현

수작업으로 모든 자원을 배치하면 수정과 확장이 어렵기 때문에, 인스펙터에서 프리팹을 추가하면 높이 구간에 따라 자동으로 자원이 배치되는 구조를 구현했습니다. 이를 통해 맵 수정이나 자원 추가 작업을 더 쉽게 할 수 있도록 개선했습니다.

침수 상태와 UI 연결

물이 차오르는 시스템만으로는 플레이어가 위험 상태를 직관적으로 인식하기 어려웠습니다. 플레이어가 물에 잠겼을 때 화면 패널, 파티클, 데미지, 체력 UI가 함께 반응하도록 연결하여 침수 상태를 시각적으로 전달했습니다.

아이템 획득과 사용 흐름 통합

아이템 획득, 인벤토리 저장, 퀵슬롯 선택, 손 장착, 소비 효과 적용이 따로 동작하면 실제 플레이 흐름이 끊길 수 있었습니다. 이를 하나의 흐름으로 연결하여 플레이어가 아이템을 자연스럽게 사용할 수 있도록 구성했습니다.

Result & Improvement

The Sinking Island에서는 인벤토리, 퀵슬롯, 아이템 사용, 수위 상승, 침수 UI, 체력 UI, 사운드 시스템, 자원 및 몬스터 자동 배치 시스템을 구현했습니다.

향후 개선 방향으로는 몬스터 종류를 문자열 이름이 아닌 enum 또는 별도 데이터 컴포넌트로 관리하고, 자원 스폰 역시 더 많은 오브젝트를 처리할 수 있도록 공간 분할 방식으로 개선할 수 있습니다.
